# Site

Набор статических HTML-страниц. Публикуется через **GitHub Pages**.

## Страница ЧМ-2026 — `wc2026.html`

Адаптивная страница чемпионата мира 2026: группы, сетка плей-офф и прогноз.
Открывается по адресу `https://<ваш-логин>.github.io/<репозиторий>/wc2026.html`.

### Что внутри
- **Мобильная вёрстка** — карточки групп в одну колонку на телефоне, тянущиеся вбок
  табы и сетка плей-офф со свайпом, учёт «чёлки» (safe-area), тап-таргеты от 44px.
- **Живые результаты без ключа** — таблицы групп подгружаются из открытого API
  ESPN (`site.api.espn.com`) прямо в браузере. Регистрация и токен не нужны,
  CORS открыт — работает из коробки на любом статик-хостинге.
- **Кто выходит дальше — в реальном времени** — в матчах плей-офф рядом с
  командой загорается значок `проходит` (по ходу матча ведёт) или `прошёл`
  (матч завершён, в т.ч. по пенальти при равном счёте). Те же подсветки
  отражаются и в сетке плей-офф: лидирующая по ходу матча команда выделяется,
  у пары горит метка `LIVE`. Есть фильтр матчей «Плей-офф».
- **Прогноз** — сетка плей-офф, финал и блок «кто победит» считаются прогнозом и
  хранятся в самом файле (никакой API их не отдаёт).

### Как это работает
Ничего настраивать не нужно — открыли страницу, и таблицы групп обновляются
сами (раз в минуту). Строка статуса над таблицами показывает режим:
🟢 «в реальном времени · ESPN» или ⚪ «снимок данных», плюс время обновления и
кнопку «Обновить».

Если ESPN временно недоступен или ещё не отдаёт таблицы турнира, страница
показывает встроенный снимок и честно об этом пишет — ничего не ломается.

> ESPN — неофициальный (hidden) API: бесплатный, но без гарантий и может
> измениться без анонса. Поэтому запрос завёрнут в обработку ошибок с откатом
> на снимок. Параметры live-режима — в блоке `const LIVE = {` в `wc2026.html`
> (код лиги `fifa.world`, интервал обновления).

## Публикация (GitHub Pages)
Settings → Pages → Source: ветка `main`, папка `/ (root)`. Файл `.nojekyll`
уже в репозитории — он отключает обработку Jekyll и отдаёт файлы как есть.
